### Navigator bug — root-caused + fixed (was "empty box on real iEEG")
Diagnosis (verified, not guessed): canvas correctly sized (1278×93) but `drawnPixels:0` and `scales.x={min:null,max:null}`. uPlot's constructor runs `setData(data, resetScales=false)` (uPlot.cjs.js:6078), so x is never auto-ranged → no point maps to a pixel → nothing paints (axis included). The old rAF only called `setSize()` on a width change, so ranging never fired; a same-size `setSize()` is a uPlot no-op (why the prior "forced resize" didn't help). Any later `setSize()` re-stages x as `AUTOSCALE → snapNumX(null,null) → [null,null]`, racing/re-blanking a naively-ranged chart.

Fix (504f597), mirroring TimeseriesSliceView: range x explicitly post-construct AND re-assert it in the same synchronous block as every `setSize()` (ResizeObserver + rAF) so it wins the pendScales race. Live range tracked in `xRangeRef` (updated by the setScale hook) so drag-zoom survives resizes. Also finished the previously-uncommitted "navigator fills the bottom panel" layout feature: chart sizes to its flex slot (dynamic height, floored 40px).

Z-score (a50cd59): a raw cross-channel mean is dominated by a few loud channels (iEEG std spans ~90×). Now z-scores each channel before averaging. Extracted as pure, golden-tested `navigatorMean.ts` (`channelStats` + `zscoredCrossChannelMean`, 9 tests) per N3.

Verified on real iEEG (256ch, :5173, Playwright): `drawnPixels 0→25,532`; `scales.x null→[1.06,599.9]`; fills panel (1278×93); survives resizes (shrink→14,773 / grow→17,454).
