## Fixes shipped (uncommitted — review then commit)

### Backend correctness

- **Filtered-band overlay ran on z-scored, downsampled data.**
  `_apply_bandpass_to_slice` used to run in `_prepare_slice` *after*
  `apply_slice_request` returned — i.e. after `downsample_time_axis` and
  `zscore_offset`. The filter derives `fs` from the time spacing, so
  downsampling (which makes the spacing irregular) gave it a wrong
  sample rate. Moved bandpass *inside* `apply_slice_request`, after
  `zscore_offset` but before downsample. It now sees the true `fs` and
  the per-channel z-scored scale the frontend's band-substitution
  contract expects. `state.py` — `apply_slice_request` /
  `_prepare_slice`. Also reordered `zscore_offset` to run before
  downsample for the same windowed-data reason; `display_transforms`
  now appends (was overwriting, which dropped the bandpass tag).

- **No `sosfiltfilt` padlen guard.** Short overlay windows could exceed
  the default padlen and 500. Clamp `padlen = min(default, n-1)`.
  `state.py` — `_apply_bandpass_to_slice`.

- **MINMAX downsample distorted non-lead channels.** It chose one lead
  channel's argmin/argmax time per bucket, then gathered *every*
  channel's value at that single instant — so any channel that wasn't
  the bucket's dominant one lost its envelope (spikes vanished). Now
  each feature keeps its own true min/max; the shared timestamp is still
  anchored at the dominant channel's real sample time (F5). `state.py` —
  `downsample_time_axis`. New regression test
  `test_minmax_preserves_per_channel_extrema`.

- **Stale parquet shadowed fresh csv.** On parquet-write failure the
  fallback wrote `.csv` but left the old `.parquet`; `_read_existing`
  checks parquet first, so reads returned stale decisions. Delete the
  stale parquet on csv fallback. `routers/event_review.py` —
  `_atomic_write`.

### Frontend

- **Worker decode error hung all decodes.** `workerPool.handleError`
  rejected every in-flight request pool-wide and left the dead worker in
  rotation, so one bad payload bricked future decodes. Now rejects only
  the failed worker's requests and respawns it. Added per-pending
  `worker` tracking + a `spawnWorker()` helper. `api/workerPool.ts`.

- **`_resetWorkerPool` leaked workers** (no `terminate()`) and the
  decode-time log read `buffer.byteLength` *after* the transfer (always
  0). Added `terminate()` and captured `byteLength` before
  `postMessage`. `api/workerPool.ts`.

- **Double-fetch on first load.** `logV2PayloadSize` fired a redundant
  full v1 `getTensorSlice` per view to compute a size ratio (already
  measured). Removed the comparison fetch; keep the v2 size log.
  `api/queries.ts`.

- **Spectrogram log-axis mismatch.** Canvas fell back to linear when
  `fLo<=0`, but Y-ticks / crosshair / cursor still used log. Unified all
  on one `useLogAxis = freqLogScale && fLo > 0` predicate.
  `components/views/SpectrogramView.tsx`.

- **MaskPanel push dedup was global, not per-tensor.** Two tensors with
  identical masks (e.g. both `[1,2,3]`) skipped the push for the second.
  Keyed the dedup ref on `{tensor, key}` and seed it from hydration so a
  hydrated value isn't echoed back. `components/controls/MaskPanel.tsx`.

- **Region labels mislabeled when channel viewport scrolled.**
  `inferredNML` was derived from the visible 32-channel window, so the
  flat-id mapping (`ap*nML+ml`) broke once scrolled past the max-ML
  column. Compute it from the full (uncapped) channel set in the
  data-pipeline memo. `components/views/TimeseriesSliceView.tsx`.
