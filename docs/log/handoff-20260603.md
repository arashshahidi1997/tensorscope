## The navigator bug (diagnosed, parked)

Symptom: navigator strip renders an **empty box** on the real iEEG.
**Confirmed via headless Playwright** (not guesswork):
- Data is **good**: 795 finite, well-scaled mean values (`min −4.48, max 2.74`).
- All `/slice` requests return 200; v2 is OFF.
- Canvas is **properly sized** (~1678×93) but **`drawnPixels: 0`** — not even the
  axis. uPlot has good data + a sized canvas but never paints, and a forced
  resize doesn't fix it.
Diagnosis: a **chart lifecycle/sizing bug**, not data — the navigator is rendered
indirectly (render-prop into the `transition: height` bottom panel) and the
uncommitted height-tracking change made chart creation depend on a layout size
that isn't there at mount. This is the same fragile component flagged for the
**D3** overview redesign. Two pieces to redo when fixing it:
- The lifecycle/draw (create with a stable size, redraw once laid out).
- **Re-apply the per-channel z-score in the cross-channel mean** (I confirmed it
  works — real iEEG channel std spans ~90×, so the raw mean is dominated by a few
  loud channels; normalize each channel before averaging). Code is in this
  session's transcript / the reverted NavigatorView edit.
