# G4 ‚Äî event-locked summary view

## Why

From the spindle-review critique
(`docs/log/issue/issue-arash-20260511-182119-502518.md` G4): TensorScope
has a `triggered_average` transform but no UI surface for "show me the
mean ¬± std spindle waveform across all candidates." This is the single
most useful sanity check for a detector ‚Äî does the average look like a
spindle, or like a movement artifact? Reviewer adoption stalls without
this.

## What to build

A new view type `event_average` that, given an event stream + an
aggregation window (e.g. ¬±1 s around event onset), plots the mean ¬±
std of the raw signal aligned to event onset.

### Server

- New view-type entry in `_VIEW_REGISTRY`. Required dims: `time`.
- Slice handler accepts:
  - `event_stream_name: str` ‚Äî which event stream to align to
  - `lag_window: tuple[float, float]` ‚Äî e.g. `(-1.0, 1.0)` seconds
  - `max_events: int | None = 200` ‚Äî guard on extreme detector sets
  - `aggregate: "mean" | "median" | "snr" | "all_traces"` ‚Äî default mean
- Server walks events in stream, extracts `(t + lag_window[0],
  t + lag_window[1])` windows from the source tensor, stacks them,
  reduces per the `aggregate` policy. Returns `(lag, channel)` cube
  (or `(lag,)` for pooled mode).
- Reuse `cogpy.triggered_average`-style logic where possible; the
  registry has `triggered_average`/`triggered_std`/`triggered_snr`
  transforms already ‚Äî call them directly from the view handler so the
  reducer code lives in one place.

### Frontend

- New `EventAverageView` consuming the v2 labeled-tensor shape.
- Plots: lag on x, mean trace + ¬±std band. One line per channel
  (capped by the same N_VISIBLE pattern as the timeseries view),
  or a single pooled trace.
- Inline controls: lag-window numeric inputs, aggregate selector,
  channel-pooled toggle.
- Wire into `viewRegistry` + the slot-layout default.

### Tests

- Backend: `tests/test_event_average_view.py` ‚Äî synthetic LFP with
  injected spindles at known times; assert the mean recovers the
  spindle shape.
- Frontend: parity test on the v2 extractor (same pattern as
  `v2-arrow.test.ts`).

## Acceptance

1. With the audit spindle bundle loaded, opening the event-average
   view shows a clear oscillation in the 11‚Äì16 Hz band around lag=0
   when aligned to the spindle event stream.
2. Switching `aggregate` between mean/median/snr re-renders without a
   full reload.
3. `max_events` guard fires cleanly at >200 events (test against the
   bundle's 35 k events).

## Out of scope

- Subselecting events by review status ‚Äî that's a follow-up after G6
  (per-event tags) lands.
- Bootstrap CIs / non-parametric tests ‚Äî defer to v0.1.
- Sortable per-event raster underneath the average (Neuroscope2-style
  "all traces stacked") ‚Äî defer.
