# Spindle-validation review workflow — feature gaps

## Context

TensorScope is solid as an *exploration* tool — DAG, pair-mode, modern
compute, live PSD all earn that. As a tool for the *reviewer* workflow
(walking through 1000 detected spindles, accept/reject/maybe each, save
the validated set) it has gaps that make it impractical for the
bread-and-butter sleep-electrophysiology use case it's pitched at.

These aren't perf problems. The v2 contract work helps none of them.
They are product gaps in the event-review surface.

## Gaps (prioritized)

### P0 — workflow blockers

**G1. No filtered-band overlay on timeseries.**
Validating a candidate spindle requires raw + 11–16 Hz bandpass on the
same axis (or paired sub-row). TensorScope's processing pipeline
(`notch`, `cmr`, `zscore`) is global and stateless to the view, not a
"show this band on top of raw for this candidate" toggle. Neuroscope2
had this as a literal toggle. Without it, validators are squinting at
envelopes they can't see.
*Acceptance:* per-view band-filter picker (Off | Spindle | Ripple | SO |
Custom). When on, the timeseries panel shows the filtered series
overlaid on (or paired with) the raw series. Switching bands re-fetches
in <500 ms.

**G2. 32-channel cap on timeseries view.**
On a 16×16 ECoG grid (256 channels), a real spindle lights up 5–30
contiguous channels. The 32-channel hard cap (audit F4) and no
vertical scroll means the reviewer can't quickly answer "which
channels participated?" — the single most common review question.
The `channelCapActive` indicator is honest but doesn't solve the
workflow.
*Acceptance:* scrollable channel viewport. N visible at a time
(default 32), j/k or wheel scroll through all channels. Sticky labels.
Works at 256-channel scale at 60 fps.

**G3. Event review ergonomics — no keyboard, no accept/reject, no counter.**
Prev/next button + table is the floor. A 1000-event review needs:
keyboard `j`/`k`/`y`/`n`/`m`, a live "x of N reviewed" counter, a
filter "show only unvalidated", a "rejected" event stream that
persists. None of this exists.
*Acceptance:* with focus on the event panel or timeseries: `j` =
next event, `k` = prev, `y` = accept, `n` = reject, `m` = maybe.
Decisions persist (localStorage v0; persisted to disk later). Counter
visible at all times. Filter dropdown for status.

### P1 — important

**G4. No event-locked summary view.**
TensorScope has the `triggered_average` transform but no UI for "show
me the mean ± std spindle waveform across all candidates, sorted by
amplitude or duration." This is the single most useful sanity check
for a detector — does the average look like a spindle or a movement
artifact?
*Acceptance:* a new view type `event_average` or a panel that, given
an event stream + an aggregation window, plots mean ± std of the raw
signal aligned to event onset. Optionally split by channel or pooled.

**G5. No detector comparison overlay.**
When spindle + ripple detectors fire within 100 ms on the same channel
that's usually broadband noise/artifact. TensorScope shows one event
stream at a time; no way to overlay two and highlight coincidences.
*Acceptance:* event panel can show 2+ streams stacked, with visual
markers for time-coincident detections (e.g. red bar joining them).

**G6. No per-event annotation layer.**
Cluster notes / event tags. "This is a beautiful example for the
methods figure" — can't be marked. No per-event metadata at all.
*Acceptance:* each event has optional `notes: string` and
`tags: string[]` editable from the event panel. Persists.

### P2 — nice-to-have

**G7. Anatomical labels invisible on spatial map.**
Shows AP/ML *grid indices* not "this electrode sits over secondary
motor cortex". Probe geometry exists as a design doc
(`probe-layout.md`) but isn't surfaced.
*Acceptance:* when a probe-layout file is present, the spatial map
renders region labels at the cell or row level.

**G8. PSD context window is decoupled from the event.**
When the reviewer clicks an event the timeseries pans but the PSD
window-size stays fixed (UI knob). For spindle confirmation the
reviewer wants PSD over the *event duration*, not the global value.
*Acceptance:* an "lock PSD to selected event" mode that auto-sets
PSD window to `[event.t - margin, event.end + margin]`.

**G9. No validation export.**
After 4 hours of review there's nothing on disk. No
`validated_spindles.parquet`. The pipeline export round-trips
transforms but not human decisions.
*Acceptance:* "Export validated events" button → emits a parquet/csv
of `{event_id, status, notes, tags, reviewer_at}` keyed by source
event stream.

## What TensorScope already delivers (don't reinvent)

- Hypnogram + brainstate overlay — state context. NS2 didn't have this.
- Live multitaper PSD — configurable, not canned.
- DAG / derived tensors — clean upgrade for raw→filtered→envelope chains.
- Spatial heatmap for grid recordings — NS2 was linear-probe-only.
- Pair-mode — drive selection from python while viewing.
- Cursor sync across panels.

## Plan

Address P0 in order: G3 (review ergonomics) → G1 (filtered overlay) →
G2 (channel scroll). Each in its own design doc + commit. P1/P2 deferred
to follow-up issues unless cheap to bundle.
