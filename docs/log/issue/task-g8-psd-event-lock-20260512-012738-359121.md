# G8 — PSD locked to selected event

## Why

From the spindle-review critique
(`docs/log/issue/issue-arash-20260511-182119-502518.md` G8): when the
reviewer clicks an event, the timeseries pans but the PSD window-size
stays fixed (UI knob). For spindle confirmation the reviewer wants the
PSD computed over the *event duration*, not whatever number sat in the
PSD knob from the last interaction.

## What to build

A "lock PSD to event" toggle. When ON and an event is selected, the
PSD live request uses
`time_range = [event.t - margin, event.t_end + margin]` instead of
`[selection.time - windowS/2, selection.time + windowS/2]`.

### Approach

- The event records in the audit bundle have a `duration_s` field (or
  `t_end`, depending on the detector — see
  `spindle_ieeg_cortex_blobs.parquet` schema).
- Frontend tracks `psdLockToEvent: boolean` in `useAppStore` (new
  field, persist not required — session-local).
- When ON + an event is selected (via `useEventNavigation`), the
  request factory `makePSDLiveRequest` uses the event's window plus
  the configurable margin (default 0.25 s).
- UI: a checkbox/toggle in the PSD inspector / sidebar.
- When OFF: existing behavior (window from the `psdWindowS` slider).

### Server

- No server changes — the existing `psd_live` slice handler already
  accepts arbitrary `time_range`. Just need the frontend to pass the
  event-derived one.

### Tests

- Frontend: when lock=on and selected event has `t=10.0, duration=0.5`,
  the request fired has `time_range = [9.75, 10.75]` (with default
  margin).
- When lock=off the request reverts to the cursor-centered window.
- Edge case: event with no `duration` field — fall back to cursor
  window plus the configured `psdWindowS`.

## Acceptance

1. Toggle on → PSD heatmap visibly re-renders with the event's window.
2. Cursor pan while lock=on does NOT change the PSD time range
   (only event selection does).
3. Toggle off → behavior reverts.
4. Visually, the PSD heatmap window edges align with the timeseries
   event markers when locked.

## Out of scope

- Server-side event-duration extraction (the bundle event records
  already carry duration). Defer until a detector that produces only
  `t` (no duration) needs special handling.
- Animated transitions when toggling — instant cut is fine.

## References

- Existing PSD live request factory: `makePSDLiveRequest` in
  `frontend/src/api/queries.ts`
- Existing event nav: `useEventNavigation`
