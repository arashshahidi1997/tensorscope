# Extend tensorscope cogpy integration

Bundled task: broaden cogpy coverage in the transform registry, align
request/processing DTOs with cogpy signatures, then run an end-to-end
debug sweep and fix whatever breaks.

## Part 1 — Transform registry extensions

Expose these cogpy modules as TensorScope transforms (in
`src/tensorscope/core/transforms/builtins.py` + `TransformRegistry`).
Pick each module's primary user-facing function(s); wire through the DAG
executor; add a slice/view only if the output shape demands one.

- `cogpy.detect` — event/spike detection
- `cogpy.wave` — wave / propagation analyses
- `cogpy.brainstates` — beyond the current overlay + intervals API
- `cogpy.triggered` — trigger-locked averages
- `cogpy.burst` — burst detection

For each transform:
- Add a `ProcessingParamsDTO` entry (or a dedicated DTO if it doesn't fit
  the flat processing pipeline) with defaults that mirror cogpy's.
- If the output is a new tensor shape, register a view descriptor in
  `frontend/src/registry/viewRegistry.ts` and add a minimal renderer.
- If it's a DAG-only transform (no direct view), just register it.

## Part 2 — DTO alignment with cogpy signatures

Audit `src/tensorscope/server/models.py`:

- Walk every field of `ProcessingParamsDTO` and `TensorSliceRequestDTO`.
  Cross-check against the cogpy function each field drives
  (`bandpassx`, `notchesx`, `cmrx`, `zscorex`, `median_spatialx`,
  `psd_multitaper`, …).
- Rename fields to match cogpy kwarg names where divergence is cosmetic.
- Align default values with cogpy defaults.
- Surface `psd_multitaper` params (`NW`, `fmax`, `n_tapers`, etc.) cleanly
  in `psd_params` — drop any that tensorscope fabricated and doesn't
  actually forward.
- Remove DTO fields that map to nothing in cogpy (or flag them if they
  are tensorscope-only and should stay).

## Part 3 — End-to-end debug sweep

After parts 1 and 2:

1. `pixi run demo-data` (regenerate `data/demo_lfp.nc` if missing)

## Outcome

### Transforms added (`src/tensorscope/core/transforms/builtins.py`)

All registered in `BUILTIN_TRANSFORMS` and wired through the existing DAG
executor. No new view descriptors were added; these are DAG-only (output
tensors flow into existing views when their dims match).

- `cmr` — `cogpy.preprocess.filtering.cmrx`
- `notch` — `cogpy.preprocess.filtering.notchesx`
- `spatial_median` — `cogpy.preprocess.filtering.median_spatialx`
- `zscore` — `cogpy.preprocess.filtering.zscorex`
- `psd_multitaper` — `cogpy.spectral.psd.psd_multitaper`
- `psd_welch` — `cogpy.spectral.psd.psd_welch`
- `restrict_intervals` — `cogpy.brainstates.intervals.restrict`
- `perievent_epochs` — `cogpy.brainstates.intervals.perievent_epochs`
  (produces `(event, ..., lag)` tensors)
- `triggered_average` / `triggered_std` / `triggered_median` / `triggered_snr`
  — consume `(event, ..., lag)` tensors from `perievent_epochs`

### Event detectors added (`src/tensorscope/core/events/detectors.py`)

cogpy.detect and cogpy.burst produce event catalogs, not tensors — they
fit the `EventDetector` framework rather than `TransformRegistry`.

- `cogpy_ripple` — `cogpy.detect.RippleDetector`
- `cogpy_spindle` — `cogpy.detect.SpindleDetector`
- `cogpy_burst` — `cogpy.detect.BurstDetector`
- `cogpy_threshold` — `cogpy.detect.ThresholdDetector`

Each wrapper converts the cogpy `EventCatalog.df` to TensorScope's
`EventStream`, synthesising `event_id` values if the source catalog doesn't
already carry them.

### DTO changes (`src/tensorscope/server/models.py`)

- New `PsdParamsDTO` replaces the raw `dict[str, float]` on
  `TensorSliceRequestDTO.psd_params`. Fields mirror
  `cogpy.spectral.psd.psd_multitaper` kwargs verbatim (`NW`, `K`, `fmin`,
  `fmax`, `detrend`) with cogpy defaults.
- `fmax` default flipped from the legacy 100 Hz to `None` (Nyquist) to
  match cogpy. Frontend already sends an explicit `fmax`, so this only
  changes behaviour for callers that omit `psd_params`.
- `ProcessingParamsDTO` is already a thin shell over cogpy kwargs
  (`bandpass_*`, `notch_*`, `spatial_median*`, `zscore*`, `cmr`). Defaults
  already match cogpy's. No renames were required.
- `apply_slice_request` in `state.py` now forwards all five multitaper
  kwargs (previously only `NW` + `fmax`).

### Wave / burst modules not wrapped

`cogpy.wave.*` and `cogpy.burst.detect_hmaxima` / `merge_duplicate_bursts`
return pandas DataFrames mid-pipeline and don't fit the tensor → tensor
transform contract. The burst surface is covered via `cogpy_burst`
(wrapping `cogpy.detect.BurstDetector`, which itself wraps
`detect_hmaxima`).

### Ran / not ran

- Added 10 new unit tests in `tests/test_transforms.py`
  (`TestCogpyTransforms`, `TestCogpyDetectors`). They exercise each new
  transform end-to-end through the executor + cache + registry, plus a
  detection smoke test on a 1-D signal with an injected deflection.
- **`pixi run test` / `pixi run frontend-test` / UI browser sweep were

  `noverlap=-1`, all treated as "use cogpy default" in the compute

### Bugs fixed during the sweep

1. **Route ordering in `server/routers/events.py`** — `GET /events/{name}`
   was declared before `GET /events/detectors`, so `/detectors` was
   captured as a stream name and 404'd. Moved the specific `/detectors`
   and `/detect` routes above the parameterized `/{name}` / `/{name}/window`
   routes.
2. **Missing `fs` attr in cogpy detectors** — `cogpy_ripple`,
   `cogpy_spindle`, `cogpy_burst` all failed with `"Input must have a .fs
   coordinate/attribute"` because the wrappers passed raw
   `xr.DataArray`s through to cogpy. Added a local `_ensure_fs` helper in
   `core/events/detectors.py` (mirrors the one in `transforms/builtins.py`)
   and call it in all four cogpy detector wrappers.

### Known demo-data limitation (not a bug)

- `cogpy_ripple` on the demo tensor errors with `"Wn[0] must be less than
  Wn[1]"` — the demo signal is sampled at a rate below Nyquist for the
  ripple defaults (100–250 Hz). Users running on real hippocampal
  recordings at ≥500 Hz are fine. Not fixed.
