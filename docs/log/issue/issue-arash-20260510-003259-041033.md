**Severity:** medium — needed for any cohort-wide audit. Edge artefacts and bad channels currently force a re-bundle to exclude.
**Status:** open.

## What

Add a sidebar widget for masking channels (or whole AP rows / ML columns / arbitrary user-defined groups) on the active grid tensor. Mask propagates to all views via the existing slice path:
- Timeseries: masked channels drop from the z-score-offset stack.
- Spatial map: masked cells render as a distinct "masked" colour (not the value colour-mapped, not transparent — visually obvious).
- PSD live / spectrogram live: masked channels excluded from per-freq mean and spatial heatmap.
- Events: optional filter "hide events on masked channels" for stream views (toggle).

## Why this matters for the workflow

iEEG audit on sub-01/ses-04 (and every Npx run): edge channels of the 16×16 grid frequently fire 3× the interior median due to surface artefacts, bonewax tracks, etc. Spindle detector h-maxima then concentrates on those edges. The user spotted that pattern in 30 s on the spatial map — but currently can't say "exclude the four edge rings, re-render everything" without:

1. Building a new tensor with rows/cols dropped.
2. Re-running the detector on the masked tensor.
3. Re-bundling and relaunching tensorscope.

Round-trip: ~30 minutes per mask hypothesis. With this feature it's a one-click toggle.

## UX sketch

- New collapsible section in the Selection sidebar tab: **"Channel mask"**.
- Reuses the `ChannelGridRenderer` (the same heatmap component used by SpatialMap), but in "mask editor" mode:
  - Click a cell → toggle that cell.
  - Drag-select rectangle → toggle a block (with shift to ADD, alt to REMOVE).
  - Buttons: "All", "Interior" (drop the 4 edge rings — `interior_radius` slider), "None", "Invert".
  - Keyboard: `m` while hovering a cell to toggle.
- Live count display: "Masked: 96 / 256 channels".
- Mask persists in the Zustand store + an SSE-broadcast field on the server (so a paired agent can also read / write it via `pairing.PairContext.set_mask(...)` later — Phase B).

## Suggested implementation

**Server:**

- Add `channel_mask: list[int] | None` to `TensorScopeState` and `StateDTO`. List of channel ids to *exclude* (so an empty list means "everything visible").
- Per-tensor masks via `dict[tensor_name, set[int]]` if we want the mask to be tensor-specific (probably the right choice — masks transfer poorly across tensor types). Start simple with a single global mask on the active tensor; extend later.
- New endpoint `PUT /api/v1/mask` body `{tensor: str, masked_ids: list[int]}` → updates state, publishes `mask_changed` SSE.
- `apply_slice_request` consults the mask:
  - For `(time, AP, ML)` data: convert masked_ids → (ap, ml) pairs → set those cells to NaN before downstream operations. Most reductions (`mean`, `median`, `std` with `np.nanmean`-friendly variants) handle NaN correctly. For `psd_multitaper` and `mtm_spectrogram`, NaN propagates — replace with 0 for the per-channel pre-FFT step, then mark output as NaN.
  - For `(time, channel)` data: `xr.where(channel.isin(unmasked_ids), data, np.nan)` or simply `isel(channel=unmasked_ids)` if we want to drop them entirely (faster, but breaks fixed-position heatmaps — keep NaN-mask for spatial views, drop-mask for timeseries).

**Frontend:**

- `useMaskStore` (or extend `useSelectionStore`): `{maskedIds: number[], setMask, toggleMaskId}`.
- New `MaskPanel.tsx` in `components/controls/` with the renderer-driven editor described above.
- Slice request builders include `channel_mask` (or `masked_ids`) when non-empty. Cache key picks it up automatically; queries refetch when mask changes.
- SpatialMap renderer paints masked cells in a distinct colour — extend `ChannelGridRenderer.render` to accept a `maskedIds: Set<number>` param.

**Pairing API (Phase B, defer):**

- `PairContext.set_mask(masked_ids: list[int])` and `read_mask()`.
- `add_mask_preset(name, masked_ids)` for reusable masks (e.g. "edge ring", "device tracks").

## Persistence considerations

- Mask state should persist across browser refreshes (localStorage on the active tensor).
- Should it persist across `--pair` server restarts? Probably not by default — the mask is workflow state, not data state. A "save mask to sidecar parquet" button would handle the rare case where the user wants to persist for the next session.

## Out of scope

- Mask blending (e.g. soft-mask via per-channel weights). Boolean only for v1.
- Per-event-stream channel filtering driven by mask (covered by the optional toggle above; full per-stream UI is a separate feature).
- Mask presets shipped from a server-side library (e.g. "all electrode-edge artefacts" pre-filtered). Could be added later via `add_mask_preset`.
