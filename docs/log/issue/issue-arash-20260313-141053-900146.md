# issue
## Issues

### 1. PSD Spatial Map Grid Size and Labeling
- PSD spatial map displays as 8x8 instead of the expected 16x16 (demo2 data is 16x16).
- Time series spatial map correctly shows 16x16.
- Both spatial maps (time series and PSD) should have AP (anterior-posterior) and ML (medio-lateral) indices:
    - AP: top-to-bottom, 0–15 (rows).
    - ML: left-to-right, 0–15 (columns).
- Labels for AP and ML should be present and consistent on both views.

### 2. PSD and Transform Settings in Explorer Mode
- PSD settings are shown in the explorer side panel, but other transforms (e.g., spectrogram) should also have their own settings sections when added.
- PSD is currently treated as a default transform; clarify if it should appear as a node in the DAG (transform graph) or remain a quick measure on the signal tensor.

### 3. Time Series View: Y-Axis and X-Axis Interaction
- Y-axis scroll zooms in/out; want shift+scroll to adjust gain (amplitude scaling) directly, without needing to press a button.
- Y-axis drag icon is misleading; should reflect actual gesture (scroll for zoom, drag for pan).
- Click-and-drag should work for both Y-axis (amplitude pan) and X-axis (time pan), but not on the signal itself—only on the axes.

### 4. Time Selection and Navigation
- Add a time selection input next to the time scale (bottom left of time series view) for direct time entry and jump.
- Add a horizontal time slider below the time series view for panning/navigating through the signal.

### 5. Sidebar Layout
- Left sidebar is too narrow; some text (e.g., bandpass filter order) overflows. Increase width or fix layout to prevent text overflow.

### 6. Transform Parameter Input Usability
- In transform settings (e.g., spectrogram), decimal points cannot be typed directly (e.g., entering ".95" is difficult).
- Improve number input to allow intuitive entry of decimal values.
