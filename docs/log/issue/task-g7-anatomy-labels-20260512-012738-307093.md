# G7 — anatomical labels on spatial map

## Why

From the spindle-review critique
(`docs/log/issue/issue-arash-20260511-182119-502518.md` G7): the spatial
map shows AP/ML *grid indices*, not "this electrode sits over secondary
motor cortex." Sleep-spindle anatomy matters — frontal vs. parietal
spindles are different beasts. Without anatomical labels the reviewer
can't reason about the result.

The `docs/design/probe-layout.md` proposal already defines the data
model (a `ProbeLayout` sidecar on `ServerState` with per-electrode
region annotations). This task wires it through to the UI.

## What to build

Surface electrode-anatomy labels on the spatial-map view and (lightly)
on the timeseries channel labels.

### Server

- If the dataset bundle includes a `probe_layout.{json,yaml}` sidecar
  (per the probe-layout design doc), load it into `ServerState` and
  expose via `/api/v1/probe_layout` returning a `ProbeLayoutDTO`:
  - `n_channels: int`
  - `electrodes: [{channel_id, ap, ml, region: str, label: str | None}]`
- Existing `_VIEW_REGISTRY` entries don't change. Region info is
  separate from the slice payload.
- Stub `ProbeLayout` loader from `docs/design/probe-layout.md`
  §"Sidecar format". If the design doc is incomplete, file a follow-up
  and ship the trivial loader (region field per electrode).

### Frontend

- New `useProbeLayoutQuery()` in `queries.ts`.
- `SpatialMapSliceView` / `SpatialRenderer`: when probe-layout is
  present, draw region labels at the cell or row level (e.g. column
  shading by region, hover tooltip showing region name).
- `TimeseriesSliceView`: existing channel label strip (drawn in the
  uPlot `drawAxes` hook) appends `"ch5 · M2"` instead of `"ch5"` when
  a region is known.

### Tests

- Backend: load a synthetic probe-layout fixture, assert the
  `/probe_layout` endpoint shape.
- Frontend: SpatialRenderer test with mock layout → region overlay
  paint count matches expected.

## Acceptance

1. With the audit bundle + a hand-written probe-layout sidecar, the
   spatial map renders distinct region shading.
2. Hover any cell → tooltip shows region name.
3. Timeseries channel labels show `"ch5 · M2"` style labels.
4. Datasets WITHOUT a probe-layout sidecar render exactly as before
   (no breakage, no empty-state warning).

## Out of scope

- 3D / surface-rendered anatomy. v0 is text labels + region color
  shading only.
- Editing region assignments from the UI — file management lives in
  the dataset bundle.
- Inferring regions from MNI coords / atlas lookup — defer.
