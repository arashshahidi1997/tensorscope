# idea
## Event Detection Framework

### Problem
TensorScope needs to support event detection algorithms (burst detection, threshold crossing, spectral bumps, band power RMS) that identify discrete events across N-dimensional tensors. Currently, the transform DAG only handles data transformations, not event discovery.

### Questions
- Do we need a distinct "Event Detector" category separate from Transforms?
- How should event detectors integrate with the existing transform DAG and `EventRegistry`?
- Should detectors output event tables, sparse coordinates, or both?
- Can detectors work on arbitrary tensor dimensions (time, frequency, space)?

### Potential Approaches

1. **New DAG Node Type**: Extend `TransformRegistry` with `EventDetectorNode` that produces events rather than tensors
2. **Specialized Transform Output**: Allow transforms to emit structured `EventOutput` (coordinates + metadata)
3. **Post-Processing Pipeline**: Separate event detection stage after core transforms
4. **Cogpy Integration**: Port Cogpy's `burst.detect_H_maxima` and threshold detectors as reusable specs

### Event Detector Types (Priority)
- **Spectral**: H-maxima (bursts/bumps in spectrogram), PSD peaks
- **Temporal**: Threshold crossing, RMS band power
- **Spatial**: Propagation wavefronts, source localization
- **Multi-dim**: Joint time-frequency-space patterns

### UI/UX Considerations
- Pipeline tab: visually distinguish detectors from transforms
- EventTable view: show detected events with filter/drill-down
- Overlay: highlight detected events on timeseries/spectrogram/spatial views
- Parameters: expose detector config (threshold, window size, dimension targets)

### Implementation Steps
1. Abstract event detector interface (input tensor → event coordinates/attributes)
2. Port Cogpy burst and threshold detectors
3. Extend DAG schema to support event nodes
4. Add event detector UI in Pipeline tab
